Research

Prioritizing SMT Bug reports by Learning from Historical Patches

This winter as part of my Honours Project at McGill, I will be working training a Machine Learning model to classify bug reports according to priority. I will be doing this for the common SMT Z3 and CVC4.

After early success scraping bug report data from Github issues and using it to train an RNN to classify bug reports as high or low priority based on a heuristic classification based on metadata, e.g. bug reports that go unaddressed for 6 months are probably not a priority, I was able to achieve testing accuracies in the 69-75% range, which were higher than those achieved during training!

My next step is to able to extract insights from the code itself. To accomplish this I will be compiling Z3 and CVC4 with a custom llvm pass that inserts a call to a logging function that runs before every single function call and logs what function was called and with what parameters. My goal here is to then be able to feed these logs into a neural network to aid in classification by providing more context to the bug report.

I managed to successfully implement the logging as you can see on my Github. However my current approach has a flaw simply starting z3 and closing it immediately produces a 9000 thousand line long file, and solving a simple SMT instance runs for more than 15 minutes and yields a multi-gigabyte log file. This clearly is untennable for longer programs. As a solution we're looking into smarter logging.

Many of the things logged aren't terribly likely to be useful such as constructors or minor functions. So the plan from here is to generate a call graph of all plausible calls from a static analysis, next I will traverse the graph with BFS generating a list of calls above a certain depth, likely 5ish. These will be the only calls I log going forward.

The project is now done! You can see the full report and presentation here: Report and Presentation

Image recognition

This was a group project I did for COMP 551: Applied Machine Learning at McGill University.

The problem: Given an image containing up to 5 MNIST digits, recognize the order they're in and any blanks. For example, if the image contained "123" the correct output would be "1231010" where "10" symbolizes a blank space.

The issues: Training a neural network directly to do this impractical due to the multitude of possible outputs. To address this I employed an image segmentation system using scikit-learn. I wrote code to break the image into the digits that comprise it so that they can be individually recognized.
